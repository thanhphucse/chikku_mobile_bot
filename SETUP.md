# Chikku Mobile — Setup Guide

## Prerequisites

| Tool | Required For |
|------|-------------|
| Node.js 18+ | npm packages |
| Android Studio + JDK 17 | APK build |
| Xcode 15+ | IPA build (Mac only) |
| Apple Developer Account | IPA / real device |

---

## Step 1 — Configure API URL

Edit `www/js/config.js` and set your server IP:

```js
apiBaseUrl: 'http://YOUR_SERVER_IP:8000',
```

> The mobile device must be on the same Wi-Fi network as the saigonbot server.

---

## Step 2 — Install dependencies

```bash
cd chikku_mobile
npm install
```

---

## Step 3 — Add platforms

```bash
# Android
npx cap add android

# iOS (Mac only)
npx cap add ios
```

---

## Step 4 — Sync web assets to native projects

```bash
npx cap sync
```

Run this every time you change files in `www/`.

---

## Step 5 — Build APK (Android)

```bash
npx cap open android
```

In Android Studio:
- `Build → Generate Signed Bundle / APK`
- Choose APK → follow signing steps
- Output: `android/app/build/outputs/apk/`

**Or build debug APK directly:**
```bash
cd android && ./gradlew assembleDebug
```
Output: `android/app/build/outputs/apk/debug/app-debug.apk`

---

## Step 6 — Build IPA (iOS)

```bash
npx cap open ios
```

In Xcode:
- Select your Team under Signing & Capabilities
- `Product → Archive`
- Distribute App → Ad Hoc / App Store

---

## Permissions added automatically by Capacitor

| Platform | Permission |
|----------|-----------|
| Android | `RECORD_AUDIO`, `INTERNET` |
| iOS | `NSMicrophoneUsageDescription`, `NSSpeechRecognitionUsageDescription` |

---

## Project Structure

```
chikku_mobile/
├── www/                  ← Web app (edit this)
│   ├── index.html
│   ├── styles.css
│   └── js/
│       ├── config.js     ← API URL & TTS settings
│       ├── api-client.js ← HTTP calls to saigonbot
│       └── app.js        ← Main app logic
├── capacitor.config.json
├── package.json
├── android/              ← Generated by: npx cap add android
└── ios/                  ← Generated by: npx cap add ios
```

---

## STT Notes

- **Android**: Uses Web Speech API (Chrome WebView) — works out of the box
- **iOS**: Uses `@capacitor-community/speech-recognition` plugin (already in package.json)
  - After `npm install` + `npx cap sync`, the plugin is compiled into the native iOS project automatically
